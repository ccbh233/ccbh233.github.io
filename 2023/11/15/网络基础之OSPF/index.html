开放路径最短优先

开放式最短路径优先 ( Open Shortest Path First, OSPF ) ，是一种动态路由协议，因其出色的表现，目前被广泛使用。

OSPF 基础

Router ID

相当（但不同）于路由器的名字，用来标识路由器。每个路由器都不能相同。

Area

区域是从逻辑上将设备划分为不同的组，每个组用区域号 ( Area ID ) 标识。

OSPF 的区域 ID 是一个 32 Bit 无符号整数。他有两种表示方法：

  • 十进制：Area 1

  • 点分十进制：Area 0.0.0.1

两种表示方式等价。

路由器类型

  • 骨干路由器 ( Backbone Router, BR ) ：至少有一个接口属于骨干区域。

  • 区域内路由器 ( Internal Router, IR ) ：所有接口都属于同一个区域。

  • 区域边界路由器 ( Area Border Router, ABR ) ：有接口在骨干区域，有接口在非骨干区域。

  • 自治系统边界路由器 ( AS Boundary Router, ASBR ) ：引入外部路由信息。

报文类型

  • Hello：周期性发送，用来发现和维护邻居关系。

  • Data base Description ( DD, DBD ) ：描述本地LSDB ( Link State Database ) 的摘要信息，便于两台设备进行数据库同步。

  • Link State Request ( LSR ) ：邻居关系建立后，用于向对方请求所需 LSA。

  • Link State Update ( LSU ) ：用于向对方发送 LSA，即回复对方的 LSR 请求。

  • Link State Acknowledgment ( LSAck ) ：用于对收到的 LSA 进行确认。

邻居状态

在 OSPF 中，邻居关系和邻接关系是两个不同的概念。

  • 邻居 ( Neighbors ) ：OSPF 启动后，会通过 OSPF 接口本地所有运行 OSPD 协议的设备以组播 ( 224.0.0.5 ) 的形式发送 Hello 报文以告知自己的存在。如果设备接收到了来自对方的 Hello 报文，且根据报文判断条件满足，则建立邻居关系，并将所有邻居关系记录到邻居表中。Hello 报文中携带本身 RID 与自身邻居的 RID。

  • 邻接 ( Adjacencies ) ：建立邻居关系后，若条件不满足深入关系的建立条件，则两台设备稳定在邻居关系，双方周期发送 Hello 报文保活。若条件满足，则开始建立邻接关系。

邻居状态和邻接状态是通过 OSPF 状态机表现的，OSPF 共有 8 种邻居状态机。

  • Down：初始状态。

  • Attempt：对端在邻居失效时间间隔 ( Dead Interval ) 超时前未回复本端发送的 Hello 报文时进入此状态。此状态下，路由器依然每发送轮询 Hello 报文的时间间隔 ( Poll Interval ) 向对端发送 Hello 报文。该状态仅发生在 NBMA 网络中。

  • Init：收到 Hello 报文后进入此状态。

  • 2 - Way：收到的 Hello 报文中包含有自己的 RID，则进入此状态，同时标志邻居关系建立。若不需要形成邻接关系则稳定在此状态。

  • Exstart：进入 2 - Way 状态后若需要形成邻接关系则进入此状态。开始协商主从关系并确定 DD 报文序列号。

  • Exchange：主从关系协商完后进入此状态，开始交换 DD 报文。

  • Loading：DD 报文交换完成后进入此状态。

  • Full：LSR 重传列表为空后稳定在此状态。

网络类型

OSPF 网络类型是一个非常重要的接口变量，这个变量将影响 OSPF 在接口上的操作，例如采用什么方式发送OSPF协议报文，以及是否需要选举 DR、BDR 等，接口默认的OSPF网络类型取决于接口所使用的数据链路层封装，OSPF 支持4种网络类型。

  • P2P ( Point-to-Point, 点到点 )

  • P2MP ( Point to Multi-Point, 点到多点 )

  • BMA ( Broadcast Multiple Access, 广播式多路访问 )

  • NBMA ( Non-Broadcast Multiple Access, 非广播式多路访问 )

载波侦听多路访问

CSMA（Carrier Sense Multiple Access，载波侦听多路访问）是一种常见的访问协议，用于在共享媒体上进行数据传输。它被广泛应用于以太网等局域网和广域网中。

CSMA协议的基本原理是，在发送数据之前，设备会先侦听网络上是否有其他设备正在发送数据。如果检测到有其他设备在发送数据，当前设备会等待一段时间，直到网络空闲为止。一旦网络空闲，设备才会开始发送数据。

CSMA协议仅依靠”载波侦听”的机制来判断网络是否处于忙碌状态。如果设备在发送数据之前侦听到了载波信号，即表示网络正在被占用，设备会等待一段时间再尝试发送。然而，由于多个设备可能同时开始发送数据，这就可能导致碰撞（collision）的问题。当多个设备在同一时间尝试发送数据时，它们的信号会相互干扰，导致数据损坏。

通过CSMA协议，设备可以按照一定的规则共享网络资源，避免数据冲突和碰撞，从而提高网络的传输效率和公平性。随着技术的进步，CSMA协议也发展出了不同的变体，如非坚持CSMA、1-坚持CSMA等，以满足不同场景下的需求。

在网络负载较重的情况下。CSMA仍然会发生数据碰撞。而 CSMA/CD 在发送数据之前不仅会侦听网络，而且还会检测网络上是否存在碰撞信号。如果存在碰撞，则会立即停止发送数据，并通过特殊的反馈机制（反弹，发送信号加强冲突）通知网络中的其他计算机停止发送数据。计算机之间将等待随机时间，然后重新尝试发送数据。

在发送帧之前，先监听信道（载波），若信道空闲则将帧完整发送出去，若信道忙则推迟发送。

载波侦听多路访问

  • 1-坚持CSMA
    信道忙不能发送数据帧时，开始一直监听信道，一旦发现信道空闲马上发送数据。

  • 非坚持CSMA
    信道忙不能发送数据帧时，不立即监听信道，而是随机等待一段时间后再侦听信道。

  • P-坚持CSMA
    有P的几率执行 1-坚持CSMA 的方案，有1-P的几率执行 非坚持CSMA 的方案。

实际上，这三种协议在发生冲突后还是会坚持把数据帧发送完，往往造成资源浪费。

载波侦听多路访问与冲突检测

CSMA/CD（Carrier Sense Multiple Access with Collision Detection，载波侦听多路访问与冲突检测）是一种用于以太网等局域网中的访问协议。

CSMA/CD协议扩展了基本的CSMA协议，增加了冲突检测机制。它的基本原理如下：

  • 载波侦听（Carrier Sense）：设备在发送数据之前，会侦听网络上是否有其他设备正在发送数据。如果检测到载波信号，表示网络正在被占用，设备会等待。

  • 冲突检测（Collision Detection）：当一个设备开始发送数据后，它会继续侦听网络，以检测是否有其他设备也在发送数据。如果发现与自己发送的数据冲突（即多个设备同时发送导致干扰），设备会立即停止发送，并发送一个干扰信号给其他设备，以通知它们发生了冲突。

  • 退避算法（Backoff Algorithm）：当冲突发生后，参与冲突的设备会等待一段随机的时间，然后再次尝试发送数据。这个随机的退避时间能够降低碰撞的概率，增加发送成功的机会。如果多次重试后仍然发生冲突，设备会停止发送并报告发送失败。

通过CSMA/CD协议，多个设备可以共享同一以太网媒体进行数据传输。当冲突发生时，协议能够迅速检测到并采取相应的措施（比如停止发送数据帧），以减少数据冲突以及资源浪费。
